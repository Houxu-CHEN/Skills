# 购买记录 PDF

把本地截图与发票整理成一份可交付的购买记录材料。默认产物是 `.tex` 源文件和编译后的 `.pdf`，结构为首页总表、逐项单页、末页 TODO。

## 适用场景

以下情况优先使用这个 skill：

- 用户给的是本地文件，而不是口述台账
- 材料主要由订单截图、付款截图、发票 PDF 或发票截图构成
- 用户要求按固定版式输出 PDF
- 用户强调金额核对、缺失项 TODO、可报销性

如果用户只是要做普通表格台账，不需要证据截图和发票页，不要用这个 skill。

## 工作流

1. 先读本地规则。
- 如果当前工作区或上级目录有 `AGENTS.md`，先读它
- 默认用中文输出，除非用户明确要求别的语言

2. 收集候选文件。
- 优先搜索用户指定目录，常见是 `~/Downloads`、`~/Desktop` 或当前项目目录
- 用 `rg`、`find`、`ls -lt` 先列出图片和 PDF
- 不要一上来就生成最终文档，先做配对表

3. 先建配对表，再写正式文档。
- 通过物品名、金额、时间邻近性、订单号把 `购买记录` 和 `付款记录` 配起来
- 发票优先按金额匹配，再用商品名或商家名复核
- 至少在工作笔记里维护这些字段：
  - 类型
  - 物品简称
  - 金额
  - 订单截图路径
  - 付款截图路径
  - 发票路径或缺失状态
  - 订单号
  - 备注

4. 物品命名要保守。
- 正文中的物品名，用订单截图标题的缩略
- 不要擅自补充截图里没有明确出现的信息
- 如果截图只是列表页、局部页、裁切页，要在备注或 TODO 里说明，不要装作是完整凭证

5. 默认使用以下版式，除非用户明确改。
- 第 1 页：分类汇总表
  - 按 `服装 / 道具 / 家具` 等类型分组
  - 每条包含 `物品名 / 价格 / 空备注`
  - 每个类型有小计
  - 页面末尾有总计
- 后续每个物品单开一页
  - 左上：购买记录
  - 右上：付款记录
  - 下方：发票
- 最后一页：TODO
  - 缺哪些订单截图
  - 缺哪些付款截图
  - 缺哪些发票
  - 哪些文件只是局部证据或无法直接使用

6. 严格处理金额一致性。
- 如果用户说“三个金额必须严格相等”，就不能把金额不匹配的发票硬塞进该物品页
- 发票金额与订单金额或付款金额不一致时：
  - 正文该页保留发票占位
  - 在 TODO 或备注中标记为“有票但未匹配”
- 明确区分三种状态：
  - 已匹配
  - 缺失
  - 已存在但未匹配

7. 优先用 LaTeX 保证可重复生成。
- 在项目目录里创建或更新 `.tex`
- 编译出 `.pdf`
- 如果 `tectonic` 可用，优先用它
- 如果沙箱内编译失败且像是环境或权限问题，改用提权重试
- 如果是 TeX 语法错误，先修 `.tex`，不要盲目换工具

8. 交付前必须验证。
- 首页总计必须等于所有已纳入项目金额之和
- 每个项目页都要符合用户指定构图
- 每个已嵌入发票都要么金额匹配，要么明确标成缺失或未匹配
- PDF 文件必须真实存在，并把输出路径告诉用户

## 实操规则

- 不要偷偷缩小范围。用户如果指了一大段图片范围，就应该按全量去扫，或者明确说明哪些被排除了
- 列表总览截图不能默认算独立条目，除非用户要求保留
- 像发票抬头局部、截断发票页这类材料，不算完整发票
- 如果有重复发票 PDF 且发票号相同，只保留一份，并在 TODO 或备注中说明重复情况
- 目标目录里新增了文件时，要重新扫描，不能默认沿用旧配对结果

## 常用命令

列出候选文件：

```bash
find ~/Downloads -maxdepth 1 \( -name 'IMG_*.PNG' -o -name 'IMG_*.jpg' -o -name '*.pdf' \) | sort
```

快速抽取 PDF 发票文字：

```bash
python3 - <<'PY'
from pathlib import Path
from pypdf import PdfReader
for p in Path.home().joinpath("Downloads").glob("*.pdf"):
    text = "\n".join((page.extract_text() or "") for page in PdfReader(str(p)).pages)
    print("===", p.name, "===")
    print(text[:3000])
PY
```

用 Tectonic 编译：

```bash
tectonic --outdir "<输出目录>" "<文件.tex>"
```

## 验证

至少做这些检查：

```bash
test -f "<report>.tex"
test -f "<report>.pdf"
ls -lh "<report>.tex" "<report>.pdf"
```

最后汇报时要明确说明：

- 一共纳入了多少项
- 哪些项已经匹配到发票
- 哪些项还留在 TODO

## 易错点

- LaTeX 正文里不要直接写反引号包裹的文件名，也不要裸写下划线；要改成 `\texttt{}`，并把 `_` 转义成 `\_`
- `tectonic` 在沙箱内外的失败原因可能不同，要区分“环境失败”和“TeX 语法失败”
- 有些付款截图其实是列表页，不是付款详情页；它们可以辅助配对，但证据强度弱于详情页
- 用户明确指定构图时，不要因为排版方便就改成别的版式
